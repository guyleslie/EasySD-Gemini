#==============================================================================
# CartInterface.cpp - C64 Cartridge Bus and Protocol Handling
# -----------------------------------------------------------------------------
# Purpose: Handles all low-level C64 bus interactions, including PHI2 clock
#          synchronization, /RESET control, and protocol state machine for
#          receiving commands from the C64. Manages safe bus access, data
#          latching, and error recovery.
#
# Key Architecture (2026):
#   - All bus changes are synchronized to PHI2 low for glitch-free operation
#   - Receive state machine robustly handles identifier timeouts and errors
#   - Tristate logic ensures BASIC-safe idle state for the C64
#   - All protocol transitions and errors are logged for debugging
#
# Lessons Learned (2026):
#   - Always synchronize bus changes to PHI2 low to avoid mapping glitches
#   - Stale identifier timeouts are critical for crash recovery
#   - Never leave the C64 in a non-BASIC-safe state after errors
#   - All error and state transitions must be logged
#
# See also: docs/CARTRIDGE_PROTOCOL_OVERVIEW.md, EasySD.ino, CartApi.cpp
#==============================================================================
